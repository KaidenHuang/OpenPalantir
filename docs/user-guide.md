# 使用说明 | User Guide

## 1. 安装与启动

### 环境要求

| 组件 | 最低版本 | 推荐版本 |
|------|---------|---------|
| Python | 3.10 | 3.11+ |
| Node.js | 18 | 20+ |
| Neo4j | 4.4 | 4.4.8+ |
| Redis | 6.0 | 7.0+ |

### 方式一：Docker 部署（推荐）

```bash
# 1. 克隆仓库
git clone https://github.com/your-org/OpenPalantir.git
cd OpenPalantir

# 2. 配置环境变量
cp backend/.env.example backend/.env
# 编辑 backend/.env，至少修改 NEO4J_PASSWORD

# 3. 一键启动
docker-compose up -d

# 4. 访问
# 前端: http://localhost:5175
# API 文档: http://localhost:8001/docs
```

### 方式二：手动安装

```bash
# 1. 安装 Neo4j 和 Redis
# 下载 Neo4j Community 4.4: https://neo4j.com/download/
# 下载 Redis 7: https://redis.io/download/

# 2. 后端
cd backend
cp .env.example .env
pip install -r requirements.txt
uvicorn main:app --reload --port 8001

# 3. 前端
cd frontend
npm install
npm run dev

# 4. 访问 http://localhost:5175
```

### 配置说明 (`backend/.env`)

```ini
# Neo4j (必配)
NEO4J_URI=bolt://localhost:7687
NEO4J_USER=neo4j
NEO4J_PASSWORD=你的密码

# Redis
REDIS_HOST=localhost
REDIS_PORT=6379

# LLM API 密钥 (可选，不配则使用本地 Ollama)
OPENAI_API_KEY=
SILICONFLOW_API_KEY=
DEEPSEEK_API_KEY=
```

## 2. 功能使用指南

系统包含 8 个功能标签页，从左到右依次使用构成完整的分析链路。

### 2.1 文档管理

**场景**：你有大量文档（小说、论文、企业文件），希望从中提取人物关系、事件脉络。

**操作步骤**：

1. 进入「文档管理」标签页
2. 点击「添加文档源」，选择文件或目录
3. 点击文档旁的「处理」按钮
4. 系统自动执行：
   - 解析文档内容（支持 PDF/Word/Markdown/TXT/图片OCR）
   - 生成多层级摘要树（左侧面板可浏览）
   - 提取实体（人物、组织、地点等）和关系
   - 写入图谱数据库
5. 处理后可在「实体管理」和「图谱可视化」查看结果

**支持的格式**：

| 格式 | 说明 |
|------|------|
| .txt | 纯文本 |
| .pdf | PDF 文档 |
| .docx | Word 文档 |
| .md | Markdown 文件 |
| .jpg/.png/.bmp | 图片（OCR 识别） |

---

### 2.2 数据库管理

**场景**：你已有结构化数据库（MySQL/PostgreSQL/SQLite），希望将其导入图谱进行分析。

**操作步骤**：

1. 进入「数据库管理」标签页
2. 点击「添加连接」，填写：
   - 连接名称、数据库类型（MySQL/PostgreSQL/SQLite）
   - 主机地址、端口、用户名、密码、数据库名
3. 点击「测试连接」验证
4. 连接成功后，选择目标数据库 → 点击「分析 Schema」
5. 系统会用 LLM 自动标注每列的业务含义
6. 选择需要导入的表 → 点击「导入图谱」
7. 等待异步任务完成，数据即进入图谱

**支持的数据源**：
- MySQL 5.7+
- PostgreSQL 12+
- SQLite 3

**实体命名**：数据库行实体以 `{表名}:{主键值}` 命名，如 `employees:1001`。

---

### 2.3 实体管理

**场景**：浏览、搜索、编辑系统中所有的实体。

**功能**：

- **搜索**：按名称模糊搜索，支持按类型过滤（人物/组织/地点/事件/概念）
- **查看详情**：点击实体查看其属性、关联关系
- **编辑**：修改实体名称、类型、描述、自定义属性
- **删除**：删除实体及其所有关联关系

---

### 2.4 图谱可视化

**场景**：以可视化方式探索知识图谱。

**功能**：

- **2D 力导向图**：节点=实体，连线=关系。支持拖拽、缩放、悬停高亮
- **3D 视图**：通过 react-force-graph-3d 展示立体图谱
- **交互**：
  - 点击节点查看详情
  - 点击边查看关系
  - 自由拖拽节点调整布局

---

### 2.5 分析引擎

**场景**：对图谱进行结构化分析，获取深层洞察。

#### 路径分析
- 输入：起始实体 A → 目标实体 B
- 输出：两者之间的最短路径（经过的节点序列）

#### 社区发现
- 使用 Louvain 算法自动检测图谱中的社区结构
- 输出：各社区包含的节点数、模块度评分

#### 中心性分析
- 支持多种中心性指标：
  - **度中心性**：连接数最多的节点
  - **介数中心性**：信息流动的关键"桥梁"节点
  - **特征向量中心性**：与重要节点相连的节点
  - **PageRank**：类似 Google 排名的权威节点

#### 趋势分析
- 分析实体的时间分布趋势

#### 报告生成
- 将分析结果导出为 HTML 报告

---

### 2.6 智能决策

**场景**：用自然语言提问，系统结合图谱 + 数据库 + 文档为你提供决策建议。

**操作步骤**：

1. 进入「智能决策」标签页
2. 在输入框中输入问题，如：
   - "分析当前组织中的人员分布"
   - "最近一周有哪些重要事件？"
   - "人物 A 和人物 B 之间有什么关联？"
3. 系统返回结构化回答：
   - **摘要**：简明结论
   - **态势分析**：背景和现状
   - **关键问题**：需要注意的风险点
   - **方案选项**：可选应对方案
   - **建议**：推荐方案
   - **工单**：可执行的任务清单

**领域插件**：系统支持按问题域选择合适的分析插件，当前内置 `workforce`（人力资源）领域插件。

---

### 2.7 任务管理

**场景**：查看和管理所有异步任务的执行状态。

**功能**：

- 查看所有任务（支持状态筛选：待处理/运行中/已完成/失败）
- 查看任务详情（执行日志、开始/结束时间、进度）
- 取消运行中的任务
- 删除任务记录

---

### 2.8 模型管理

**场景**：配置 LLM 模型，系统的实体提取、Schema 标注、决策推理都需要 LLM。

**操作步骤**：

1. 进入「模型管理」标签页
2. 系统预置了四个模型配置：
   - **Ollama**：本地模型（需安装 Ollama，默认 http://localhost:11434）
   - **OpenAI**：云端模型（需提供 API Key）
   - **DeepSeek**：云端模型（需提供 API Key）
   - **硅基流动**：云端模型（需提供 API Key）
3. 配置正确的 API URL 和密钥
4. 点击「测试连接」验证
5. 启用需要使用的模型

**模型类型**：
- **本地**：通过 Ollama 运行，数据不出本地，适合隐私敏感场景
- **云端**：通过 OpenAI-compatible API 调用，速度快，适合对延迟敏感的场景

## 3. 注意事项

1. **首次使用建议**：
   - 先配置至少一个 LLM 模型（推荐本地 Ollama + qwen2.5:7b）
   - 用小文档测试完整流程
   - 再导入大批量数据

2. **性能建议**：
   - 大批量文档处理建议分批进行
   - 数据库导入单表上限 1000 行
   - 图谱节点超过 10 万时建议启用分区功能

3. **数据安全**：
   - 云端模型会将数据发送到外部 API，敏感数据请使用本地 Ollama
   - `.env` 文件中的密钥请勿提交到版本控制（已加入 `.gitignore`）
